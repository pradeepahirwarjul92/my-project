Q16 - You have an array [1,2,3,4] can yu split it into 2 subsets with equals sum ? 
Ans - package challenge60.days.day05;

public class SplitIntoTwoSubsets {
	public static boolean canPartition(int[] nums) {
		int sum = 0;

		// Step 1 : calculate total sum

		for (int num : nums) {
			sum += num;
		}

		// If sum is odd, can't split equally
		if (sum % 2 != 0) {
			return false;
		}

		int target = sum / 2;

		// Step 2 : DP array
		boolean[] dp = new boolean[target + 1];
		dp[0] = true;

		// Step 3 : subset sum logic

		for (int num : nums) {
			for (int i = target; i >= num; i--) {
				dp[i] = dp[i] || dp[i - num];
			}
		}

		return dp[target];

	}

	public static void main(String[] args) {
		int arr[] = { 1, 2, 3, 4 };
		System.out.println(canPartition(arr));
	}

}

Q22 . An OTP is valid for exactly 30 seconds, even through no one stored it anywhere . How does server verify it without ever saving it?
Ans -	“The server doesn’t store OTPs because it uses a time-based one-time password algorithm (TOTP). Both client and server share a secret key. The OTP is generated by applying a cryptographic hash (like HMAC) on the secret key combined with the current time window (usually 30 seconds). During verification, the server recomputes the OTP using the same secret and current time window and compares it with the user input. Since both sides independently derive the same value, no OTP storage is required.”	

Q23 - what is actual execution order of an SQL query ? 
	(FROM, WHERE, GROUP BY, HAVING, SELECT , ORDER BY)
	SELECT department, COUNT(*) AS emp_count
FROM employees
WHERE salary > 50000
GROUP BY department
HAVING COUNT(*) > 5
ORDER BY emp_count DESC;

And - 🔍 Step-by-step meaning
	1. FROM (data source selection)
		-	Tables are loaded
		-	Joins are applied
	👉 “Where is the data coming from?”

	2.	WHERE (row filtering)
		-	Filters individual rows
		-	Happens BEFORE grouping
	👉 “Which rows are allowed?”
		
	3.	GROUP BY (aggregation formation)
		-	Groups rows into buckets
	👉 “How do we group data?”
		
	4.	HAVING (filter groups)
		-	Filters aggregated groups
		-	Works like WHERE but after grouping
	👉 “Which groups should remain?”
	
	5.	SELECT (projection)
		-	Chooses columns
		-	Computes expressions, aggregates
	👉 “What should be displayed?”

	6.	ORDER BY (sorting final result)
		-	Sorts final output
	👉 “How should results be ordered?”		

Q24 - 	Why count(*) is faster than count(column) ?
Ans - “COUNT(*) counts all rows and doesn’t require checking any column values, while COUNT(column) counts only non-null values 
and requires a NULL check for each row. Because of this, COUNT(*) can be slightly faster, especially on large tables or when the 
column contains NULLs. Additionally, databases often optimize COUNT(*) to use indexes or metadata. However, in modern databases, 
the performance difference is usually minimal as query optimizers handle both efficiently.”

Q25 - horizontal scalling vs vertical scalling ?
Ans -“Vertical scaling means increasing the power of a single server, like adding more CPU or RAM. Horizontal scaling means adding more servers and distributing the load among them.
