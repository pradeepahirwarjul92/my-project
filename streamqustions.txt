1. What is Stream API in Java 8?
Ans - I use Stream to write clean, declarative, and maintainable code for bulk data operation like filtering, transaformation, aggregation, and grouping - 
without explict iteration . 
	Technically, Stream works on pipeline model : 
	  Source - collection.stream()
	  Intermediate operation(Lazy) - filter(), map(), sorted(), distinct()
	  Terminal operations - collect(),reduce(),count(),foreach

 Key points I usually highlight :
	# Lazy evaluation improves performance
	# Support parallel processing using parallelStream()
	# Encourages immutability and side-effect-free logic
	# Works well with Collectors for grouping, partitioning, summarizing 
	

2. 	Difference between Collection and Stream?
Ans - # Collection
		. Used to store data in memory.
		. Supports CRUD operations (add, remove, update).
		. Uses external iteration (for-loop, iterator).
		. Data is available immediately.
		. Can be iterated multiple time
		
	  # Stream
		. Used to process data, not store it.
		. Supports functional-style operations like filter, map, reduce.
		. Uses internal iteration.
		. Follows a pipeline model (lazy intermediate operations + terminal operation).
		. Cannot be reused once consumed.

3. 	What is the difference between intermediate and terminal operations?	
Ans - 	Intermediate operations are lazy and optimized — operations are processed in a single pass when terminal operation is called.
		Terminal operation is mandatory; without it, stream does nothing.
		After terminal operation, the stream cannot be reused.
	
4. What happens if no terminal operation is used?
Ans 	Without a terminal operation, stream pipeline is not executed.
		No data processing happens.
		No exception is thrown.
		Stream behaves like a blueprint until a terminal operation is called.
		
5. What is lazy evaluation in Streams?
Ans - 	Lazy evaluation means intermediate operations in a stream are executed only when a terminal operation is called, 
allowing optimized and efficient single-pass data processing.	

6. Can a Stream be reused after a terminal operation?
Ans - A Stream cannot be reused after a terminal operation because it gets consumed. To process the data again, a new Stream must be created from the source.

7. What is the difference between map() and flatMap()?
Ans - map() is used for simple transformation of elements, whereas flatMap() is used when the transformation produces nested streams and we need to flatten them 
into a single stream.

8. What is the difference between filter() and distinct()?
Ans - filter() is used for conditional filtering based on business logic, whereas distinct() is specifically used to eliminate duplicate elements using equals and hashCode.

9 . What is peek() used for?
Ans -  
🔹 Purpose of peek()
	Performs an action on each element
	Does not modify the stream
	Mainly used for logging / debugging
	Lazy — executes only when terminal operation is called


 List<Integer> list = Arrays.asList(10,2,6,8,4,85,0,45,6,4,7,4,1,5,12);
    List<Integer> peekDemo =   list.stream()
               .filter(n -> n >10)
               .peek(n -> System.out.println("After filter::" +n))
               .map(n -> n*2)
               .collect(Collectors.toList());
    
        System.out.println("peek demo :: " +peekDemo);
		
10. What is the difference between forEach() and forEachOrdered()?
Ans - 	forEach() focuses on performance and may not preserve order in parallel streams, whereas forEachOrdered() guarantees encounter order even in parallel processing, 
with a possible performance trade-off.	

11 - What is Collectors class?
Ans -	The Collectors class in Stream API (introduced in Java 8) is a utility class that provides factory methods to collect 
		stream elements into different forms like List, Set, Map, or perform grouping, partitioning, and aggregation.		
		It is mainly used with the collect() terminal operation.
		
12 - What is the default Map implementation used by groupingBy()?
Ans - By default, groupingBy() uses a HashMap, but we can provide a custom Map supplier 
			like LinkedHashMap or TreeMap if ordering or sorting is required.
		
	🔥 Important Points
			Default Map → HashMap
			Order is not guaranteed

13 - How to change the Map type in groupingBy()?
Ans - To change the Map type in groupingBy(), we use the overloaded method 
	that accepts a Map supplier, allowing us to use LinkedHashMap, TreeMap, or 
	any custom Map implementation as per requirement.
	
	🔥 Why This Works
			Internally, groupingBy():
				Uses the provided Supplier to create the Map.
				Accumulates values per key.
				Uses combiner for parallel streams.
			
14 - What is a downstream collector?	
Ans - A downstream collector is a collector that is applied after grouping or partitioning, to further process the grouped elements.
	Internally

🔥 Internally
	groupingBy(classifier, downstreamCollector)
		First → classify elements by key
		Then → apply downstream collector on each group
		Instead of List<T>, you get aggregated or transformed result

15 - Difference between groupingBy() and partitioningBy()?
Ans - partitioningBy() is a specialized version of groupingBy() for boolean conditions, 
always producing two groups, while groupingBy() is more flexible and can create 
multiple groups based on any classification function.	

16 - What is counting() collector?
Ans - counting() is a downstream collector in Java 8 Stream API that counts the number of elements in a stream or within each group.
	counting() is a downstream collector used to count elements in a stream or within grouped results, returning the count as a Long value.

18 - 	What is collectingAndThen()?
Ans - collectinAndThen() is used to apply a final transformation after a collector finished its operation, 
	enabling post-processing of collected results within the same stream pipeline.
	
	🔥 Why it's Used
	- When result of collector need post-processing
	- Help avoid extra stream operation
	- Keep logic inside collect() pipeline


19 - How to make grouping result immutable?
Ans - To make the grouping result immutable, we use collectingAndThen() to wrap the result with an unmodifiable collection.	
	
		 Map<String,List<Employee>> immutableList = employees.stream()
                .collect(Collectors.collectingAndThen(
                        Collectors.groupingBy(Employee::getDepartment), 
                        Collections::unmodifiableMap
                ));
				
20 -	Difference between Comparable and Comparator?
Ans - 	🔥 Real-World Understanding
		Use Comparable when class has a single natural order (e.g., Employee by ID).
		Use Comparator when sorting varies (by salary, name, date, etc.).
			
		Comparable defines the natural ordering of a class internally using compareTo(), 
		while Comparator allows multiple custom sorting strategies externally using compare(), 
		providing greater flexibility.	
	
21 - What is Comparator.comparing()?
Ans - Comparator.comparing() is a Java 8 utility method that creates a Comparator 
	based on a key extractor function, making sorting more concise, readable, 
	and flexible compared to traditional Comparator implementations.

22 - What is Comparator.comparingLong()?
Ans - Comparator.comparingLong() is used to create a comparator based on a long 
primitive field, avoiding boxing overhead and providing better performance compared 
to Comparator.comparing() when dealing with primitive long values.

24 - How to sort by multiple fields?
Ans - 	In Java 8, multi-level sorting is achieved using Comparator.comparing() followed by thenComparing(), 
allowing clean, readable, and flexible sorting logic across multiple fields.
