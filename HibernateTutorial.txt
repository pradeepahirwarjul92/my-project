Q1 - Difference: save() vs persist()
Ans 
| save()                    | persist()    |
| ------------------------- | ------------ |
| Returns ID                | No return    |
| Works outside transaction | No           |
| Hibernate specific        | JPA standard |

Q2 - get() vs load()
Ans - 
	| get()                     | load()           |
	| ------------------------- | ---------------- |
	| Returns null if not found | Throws exception |
	| Hits DB immediately       | Lazy proxy       |
	| Safe                      | Faster           |


Q3 - update() vs merge()
Ans - 
	| update()                          | merge()      |
	| --------------------------------- | ------------ |
	| Fails if object exists in session | Safe         |
	| Reattaches object                 | Copies state |

Q4 - Lazy vs Eager Loading

Q16 - What is Cascade in Hibernate?
Ans - Cascade = Automatic propagation of an operation from parent to child entities.
When you perform an action on a parent object (save, update, delete, etc.), 
Hibernate can automatically apply the same action to related child objects.
👉 Instead of managing each entity separately, Hibernate handles them together.

🧩 Types of Cascade in Hibernate
⭐ CascadeType.ALL

Applies all operations

PERSIST, MERGE, REMOVE, REFRESH, DETACH
⭐ CascadeType.PERSIST

Save operation only

⭐ CascadeType.MERGE

Update operation only

⭐ CascadeType.REMOVE

Delete children when parent deleted

⭐ CascadeType.REFRESH

Reload child state from DB

⭐ CascadeType.DETACH

Detach children from session
